Neural Networks¶

A neural network is a parametrized network which may be defined as a artificial neural network - classical neural network - or as parametrized quantum circuits - quantum neural network. Furthermore, neural networks can be defined with respect to a discriminative or generative task.

Aqua provides an library for quantum and classical neural networks which can be used to build hybrid quantum classical AI models.

Extending the Neural Network Library

Consistent with its unique design, Aqua has a modular and extensible architecture. Algorithms and their supporting objects, such as neural networks for Artificial Intelligence, are pluggable modules in Aqua. New neural networks are typically installed in the qiskit/aqua/components/neural_networks folder and derive from the DiscriminativeNetwork class for neural networks that are supposed to perform discriminative tasks and the GenerativeNetwork class for neural networks that are supposed to perform generative tasks. Aqua also allows for Dynamically Discovered Components: new components can register themselves as Aqua extensions and be dynamically discovered at run time independent of their location in the file system. This is done in order to encourage researchers and developers interested in Contributing to Aqua to extend the Aqua framework with their novel research contributions.

Currently, Aqua supplies the following neural networks:

  • Classical Discriminator
  • Numpy Discriminator
  • Quantum Generator

Classical Discriminator¶

This discriminator is given by a PyTorch neural network. Please note that PyTorch must be installed. For installation instructions see https://pytorch.org/get-started/locally/.

The network is targeted at being used as part of the Quantum Generative Adversarial Network(qGAN) algorithm. Please refer to qGAN for further details on this algorithm. The discriminator takes an input vector where the number of represented features \(n_features \geq 1\) and outputs a label for the data sample, i.e. true/fake.

The following allows a specific form to be configured in the discriminative_network section of the Aqua Input File when the name field is set to ClassicalDiscriminator:

  • The dimension of the input vector \(n_features\):

    n_features = 1 | 2 | ...

    This parameter takes an int value greater or equal than 1. The default value is 1.

  • The dimension of the output vector \(n_out\). For a binary label this should always be set to 1.

    n_out = 1 | 2 | ...

    This parameter takes an int value greater or equal than 1. The default value is 1.

Declarative Name

When referring to the classical discriminator declaratively inside Aqua, its code name, by which Aqua dynamically discovers and loads it, is ClassicalDiscriminator.

Numpy Discriminator¶

This discriminator is given by a NumPy neural network.

The network is targeted at being used as part of the Quantum Generative Adversarial Network(qGAN) algorithm. Please refer to qGAN for further details on this algorithm. The discriminator takes an input vector where the number of represented features \(n_features \geq 1\) and outputs a label for the data sample, i.e. true/fake.

The following allows a specific form to be configured in the discriminative_network section of the Aqua Input File when the name field is set to NumpyDiscriminator:

  • The dimension of the input vector \(n_features\):

    n_features = 1 | 2 | ...

    This parameter takes an int value greater or equal than 1. The default value is 1.

  • The dimension of the output vector \(n_out\). For a binary label this should always be set to 1.

    n_out = 1 | 2 | ...

    This parameter takes an int value greater or equal than 1. The default value is 1.

Declarative Name

When referring to the classical discriminator declaratively inside Aqua, its code name, by which Aqua dynamically discovers and loads it, is NumpyDiscriminator.

Quantum Generator¶

This generator is given by a variational quantum circuit, see Variational Forms. The network is targeted at being used as part of the Quantum Generative Adversarial Network(qGAN) algorithm. Please refer to qGAN for further details on this algorithm.

The quantum generator generates outputs data samples which are fitted to a data grid. This grid is defined by min/max data values and the number of qubits \(n\) which in turn define the representation resolution.

The following allows a specific form to be configured in the generative_network section of the Aqua Input File when the name field is set to QuantumGenerator:

  • The min/max data values for data dimension \(k\):

    bounds = [[min_1,max_1],...,[min_k,max_k]]

    This parameter takes an array.

  • Given data with dimension \(k\), the number of qubits used for the representation of dimension \(j \in [1, ..., k]\):

    nm_qubits = [n_1,..., n_k]

    This parameter takes an array of length \(k\). The use of \(n\) qubits enables the representation of \(2**n\) values.

  • The generator circuit:

    generator_circuit

    The generator circuit must either be given as UnivariateVariationalDistribution for univariate data or as MultivariateVariationalDistribution for multivariate data. See Random Distributions.

  • Initial parameters used for the generator circuit:

    init_params = [param_0, ..., param_m]

    This parameter takes a 1-``dimensional ``array. The default value is None.

  • Snapshot directory, if given save intermediate parameter results to the given directory path:

    snapshot_dir = '...'

    This parameter takes a str. The default value is None.

Declarative Name

When referring to the quantum generator declaratively inside Aqua, its code name, by which Aqua dynamically discovers and loads it, is QuantumGenerator.
